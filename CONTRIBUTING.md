# Contributing

Thanks for contributing to BuildIntel.

## Collaboration Note

This project is developed with support from `Claude` and `Codex` for implementation, iteration, debugging, and QA workflows.

## Before You Start

- Open an issue or start a discussion for significant changes before beginning implementation.
- Keep pull requests focused on one feature, fix, or refactor at a time.
- Avoid mixing unrelated formatting, cleanup, and behavioral changes.

## Development Setup

```bash
npm install
npm run seed
npm run dev
```

## Pull Request Checklist

- Explain the problem and the user-facing impact.
- Add or update tests when behavior changes.
- Update docs when setup, environment variables, or workflows change.
- Confirm the app still runs locally and the affected flows were manually verified.

## Code Style

- Follow the existing project structure and naming conventions.
- Prefer small, reviewable commits with clear messages.
- Do not commit secrets, production credentials, or private customer files.
